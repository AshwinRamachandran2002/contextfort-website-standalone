1. Introduction

ContextFort Security ("we," "our," or "the extension") is a Chrome browser extension designed to monitor and record Claude AI agent mode activity for security auditing and compliance purposes.

This Privacy Policy explains what data we collect, how we use it, and your rights regarding your data. By installing and using this extension, you agree to the practices described in this policy.

Key Privacy Principles

  • All data stored locally on your device
  • No data transmitted to external servers
  • No third-party data sharing
  • You have complete control over your data
  • Data collection only during Claude agent mode

2. What Data We Collect

2.1 Web History

When Claude agent mode is active, we collect:

  • URLs visited by the agent during sessions
  • Page titles of visited websites
  • URL transition sequences (which URLs were visited in what order)
  • Timestamps of when each URL was visited
Example: If the agent navigates from gmail.com → docs.google.com → support.claude.com, we record this sequence with timestamps.

2.2 User Activity

During active agent sessions, we collect:

  • Click events (where and when clicks occurred)
  • Element metadata of clicked items (HTML tag, ID, class name)
  • Text content from clicked elements (limited to first 50 characters)
  • Click coordinates (X, Y position on screen)
  • Interaction types (click, double-click, right-click)
  • Timestamps of all interactions

2.3 Website Content

During active agent sessions, we capture:

  • Screenshots (PNG images) of the visible browser tab when agent performs actions
  • Page titles associated with screenshots
  • Visual content displayed on screen at the time of capture
Important: Screenshots capture whatever is visible on your screen, which may incidentally include text, images, videos, personal information (emails, messages, documents), private or sensitive content, and any content displayed in the browser window.

2.4 Session Metadata

We collect:

  • Session IDs (randomly generated identifiers)
  • Session start and end times
  • Session duration
  • Tab group IDs (Chrome's internal identifiers)
  • Session status (active or ended)

2.5 Configuration Data

We store:

  • URL blocking rules you configure (domain pairs that should not be navigated between)
  • Whitelist entries (URLs or hostnames you explicitly allow)
  • Extension settings and preferences

3. What Data We DO NOT Collect

We explicitly DO NOT collect:

  • Personally identifiable information (names, addresses, email addresses, phone numbers)
  • Passwords or authentication credentials
  • Credit card or financial information
  • Health or medical information
  • Location data (GPS, IP addresses, geolocation)
  • Keyboard input or keystrokes
  • Data from other extensions or browser tabs
  • Browsing activity when agent mode is NOT active
  • Background browsing or monitoring

Note: While screenshots may incidentally capture sensitive information visible on screen, we do not actively parse, extract, or analyze personal information from captured content.

4. How We Collect Data

4.1 Collection Trigger

Data collection ONLY occurs when:

  • Claude agent mode UI elements are detected (glow border, stop button)
  • Agent mode is actively running on a webpage
  • The extension detects agent interactions or navigation

Data collection DOES NOT occur:

  • During normal web browsing (when agent is not active)
  • When the extension is installed but agent mode is inactive
  • In incognito/private browsing mode (extension doesn't run there by default)

4.2 Collection Methods

  • Content scripts injected into web pages detect agent UI elements and capture events
  • Background service worker monitors tab changes and manages sessions
  • Chrome APIs used for screenshot capture (chrome.tabs.captureVisibleTab)
  • Mutation observers detect when agent UI appears or disappears

5. How We Use Your Data

5.1 Primary Purposes

Your data is used to:

  • Security monitoring: Track which websites the agent accessed
  • Audit trails: Create records of agent actions for compliance
  • Visual documentation: Provide screenshots showing what the agent did
  • Security enforcement: Apply URL blocking rules to prevent unwanted navigation
  • Session management: Organize agent activity into logical sessions

5.2 Dashboard Display

All collected data is displayed in the extension's dashboard for transparency, review, compliance, and security analysis.

6. How We Store Your Data

6.1 Storage Location

  • All data stored locally on your device using Chrome's chrome.storage.local API
  • No cloud storage or external databases used
  • No server uploads or remote backups
  • Data remains on your computer/device only

6.2 Storage Limits

  • Screenshots automatically limited to 100 most recent entries
  • Older screenshots deleted automatically when limit exceeded
  • Sessions persist until manually deleted by user
  • No automatic expiration or cleanup (except screenshot limit)

6.3 Storage Security

  • Data protected by Chrome's built-in security mechanisms
  • Only this extension can access its own storage space
  • Other extensions and websites cannot access stored data
  • Data encryption follows Chrome's storage security standards

7. How We Share Your Data

7.1 No Third-Party Sharing

We DO NOT:

  • Share, sell, rent, or trade your data with third parties
  • Transmit data to external servers or cloud services
  • Use analytics or tracking services
  • Integrate with advertising networks
  • Send data to developers or maintainers

7.2 No Data Transmission

The extension operates entirely locally on your device:

  • No network requests to external servers
  • No telemetry or usage statistics sent
  • No crash reports or error logging to external services
  • No API calls to remote endpoints

7.3 Law Enforcement

Since we do not collect, transmit, or have access to your data, we cannot provide it to law enforcement or legal authorities. All data exists solely on your device.

8. Your Rights and Choices

8.1 Access Your Data

You can view all collected data at any time:

  1. Click the extension icon
  2. Open Dashboard
  3. Navigate to Screenshots, URL Logs, or Context pages
  4. View all recorded sessions, screenshots, and URL history

8.2 Delete Your Data

You have complete control to delete data:

  • Delete All: Click "Clear All" button in Screenshots page to delete everything
  • Delete Sessions: Individual sessions can be managed through the dashboard
  • Uninstall: Uninstalling the extension removes all stored data

8.3 Control Data Collection

You control when data is collected:

  • Activate/Deactivate: Data only collected when Claude agent mode is active
  • Disable Extension: Disable the extension to stop all monitoring
  • URL Blocking: Configure rules to prevent agent from accessing specific domains
  • Whitelist: Define allowed URLs/domains for agent access

9. Data Retention

  • Indefinitely until you delete it: Data persists in local storage
  • Screenshot limit: Maximum 100 screenshots (oldest auto-deleted)
  • No automatic expiration: Sessions and URL logs retained until manual deletion
  • User-controlled: You decide when to clear data

Data is permanently deleted when you click "Clear All" in the dashboard, uninstall the extension, clear Chrome's extension data, or delete Chrome's user profile.

10. Children's Privacy

This extension is not directed at children under the age of 13 (or the applicable age of digital consent in your jurisdiction). We do not knowingly collect data from children.

11. Security Measures

11.1 Technical Security

  • Local-only storage: Data never transmitted over networks
  • Chrome security: Leverages Chrome's built-in storage encryption and security
  • No external dependencies: No third-party libraries with security risks
  • Content Security Policy: Strict CSP prevents code injection attacks
  • No remote code: All code packaged with extension

11.2 Access Controls

  • Extension-only access: Only this extension can access its stored data
  • Isolated storage: Separate from other extensions and websites
  • No cross-origin access: Cannot be accessed by web pages or other extensions

12. International Data Transfers

Since all data is stored locally on your device, there are no international data transfers. Data never leaves your computer/device, regardless of your geographic location.

17. Browser Permissions Explained

We request the following permissions for core functionality:

  • storage: Save screenshots, URLs, and sessions locally
  • activeTab: Capture screenshots of agent activity
  • tabs: Track which tabs agent opens and navigates
  • tabGroups: Organize agent tabs into visual groups
  • webNavigation: Monitor URL changes during agent sessions
  • declarativeNetRequest: Block navigation between restricted domains
  • notifications: Alert you when blocking rules are triggered
  • <all_urls>: Detect agent mode on any website (agent can run anywhere)

We only request permissions necessary for core functionality.
